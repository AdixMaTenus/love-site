  - [ ] utro.jpg — Status 200
  - [ ] kofe.jpg — Status 200
  - [ ] kino.jpg — Status 200
  - [ ] luna.jpg — Status 200
  - [ ] vmeste.jpg — Status 200

## 🆘 Если проблемы

### Проблема: "Показываются только эмодзи вместо фотографий"

**Решение:**
1. F12 → **Console**
2. Фильтруйте по `[ImageLoader]`
3. Посмотрите логи:
   - Если видите `❌ Не найдена:` для всех 5 фото → файлы не в production
   - Если видите `⏱ Таймаут загрузки` → проблема с сетью

**Что делать:**
```bash
# Проверьте что файлы действительно в public папке
ls -la d:\love-site\public\

# Должны быть:
# - utro.jpg
# - kofe.jpg
# - kino.jpg
# - luna.jpg
# - vmeste.jpg
```

### Проблема: "Консоль показывает ошибки"

**Самые частые:**

1. `❌ 404 Not Found`
   - Значит файл не в correct пути
   - Проверьте что `outputDirectory: "public"` в vercel.json
   - Перепроверьте что файлы в /public папке

2. `⏱ Таймаут загрузки`
   - Сеть медленная или CDN еще не кэшировал
   - Подождите 5 минут и перезагрузите
   - Попробуйте Hard Refresh: Ctrl+Shift+R (или Cmd+Shift+R на Mac)

3. `production: false` на Vercel
   - Это значит что дискримация по hostname не сработала
   - Проверьте что дменный имя настоящий (не localhost)

### Проблема: "На локальной машине работает, на Vercel нет"

**Проверка:**

1. **Локально:**
   - F12 → Console → должны видеть `production: false`

2. **На Vercel:**
   - F12 → Console → должны видеть `production: true`

3. **Если на Vercel видите `production: false`:**
   - Это локальная разработка на Vercel dev сервере
   - Это нормально! Пробует `public/utro.jpg` сначала
   - Если даже локальные пути не работают → проблема с путями в build

**Решение:**
```bash
# Пересоберите проект
npm run build

# Проверьте что в папке build (если есть) или public все на месте
# Если используется vercel dev:
npm run dev
```

## 🎯 Быстрая диагностика (3 шага)

1. **Открыть сайт** → F12 → Console → есть `[ImageLoader]` логи?
   - ✅ Да → идем на шаг 2
   - ❌ Нет → скрипт не загружен, проверьте путь к script.js

2. **Ищем** `✅ Загружена фотография:`
   - ✅ 5 раз видим → ВСЕ РАБОТАЕТ! 🎉
   - ❌ Нет или меньше 5 → переходим на шаг 3

3. **Смотрим на эмодзи/фотки**
   - ✅ Видим реальные фото → ИСПРАВЛЕНО! 💕
   - ❌ Видим только эмодзи → смотрите раздел "🆘 Если проблемы"

## 📞 Если ничего не помогает

1. Очистите кэш браузера: Ctrl+Shift+Del
2. Hard Refresh: Ctrl+Shift+R
3. Откройте в Incognito режиме (Ctrl+Shift+N)
4. Проверьте консоль еще раз в Device Emulation (F12 → Toggle device toolbar)

Если все еще не работает — скопируйте логи из Console и проверьте что:
- В vercel.json outputDirectory = "public" ✅
- Файлы действительно в /public папке ✅
- Имена файлов точно совпадают (с расширениями .jpg) ✅
